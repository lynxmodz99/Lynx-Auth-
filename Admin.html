How HWID Locking Works (Overview)

Step 01

User runs your app

When someone launches your software, it reads a unique hardware fingerprint from their machine — things like drive serial, CPU id, etc.

Step 02

License is checked

The app sends the key + HWID to the auth server. The server checks if this key is valid and if the HWID matches what's on record.

Step 03

Access granted or denied

If the key is valid and the HWID matches, the user gets in. If someone tries to share the key on a different machine, it gets blocked.
